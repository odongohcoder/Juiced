alerts

Alerts make it easy to display prominate information to the user. They are styled using colors from your palette or you can use the optional color, transition, background-color, and hover classes. Alerts can have an optional exit button and come in three sizes.

Add this JavaScript before your closing </body> tag for simple exit functionality: 
            <script>
            (function(){
              var _exits = [].slice.call( document.querySelectorAll('.alert-exit') );
              _exits.forEach(function(ele) {
                   ele.addEventListener("click", function() {
                       var _parent = this.parentNode;
                        _parent.className = _parent.className + " display-none";
                   });
              });
            })();
            </script>
What does this JavaScript do? It loops through all elements with a class of .alert-exit and adds an event listener on the exit buttons. Now, when an exit button is clicked, a class of .display-none is added to the parent element to hide it (so don't wrap your exit markup).

buttons

Buttons are a quick and easy way to get users clicking. Juiced makes no assumptions on how you will style your buttons, so by default there are no colors, transitions, or hover effects. Buttons are styled using the color, transition, background-color, cursor and hover classes.

cards

Cards are in. You can create them with minimal markup using .card and child .card-content block elements. They can be styled using the color, transition, background-color, cursor and hover classes.

Want stackable masonry-like cards?

Use our built in .card-masonry classes to wrap all your cards. Reponsive breakpoints are the same, so .masonry-1 means one column on mobile and .sm-masonry-2 would switch to two columns on small screens and up. Juiced supports up to 4 columns for each breakpoint.
